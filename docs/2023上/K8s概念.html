дёәд»Җд№ҲиҰҒдҪҝз”ЁpodпјҢдёәд»Җд№ҲдёҚиғҪзӣҙжҺҘдҪҝз”Ёе®№еҷЁе‘ўпјҹдҪҝз”ЁpodпјҢзӣёеҪ“дәҺдёҖдёӘйҖ»иҫ‘дё»жңәпјҢиҝҳи®°еҫ—еҲӣе»әдёҖдёӘvmпјҢеңЁvmдёҠиҝҗиЎҢеҮ дёӘиҝӣзЁӢд№ҲпјҢе…¶е®һйҒ“зҗҶжҳҜдёҖж ·зҡ„пјҢpodзҡ„еӯҳеңЁдё»иҰҒжҳҜи®©еҮ дёӘзҙ§еҜҶиҝһжҺҘзҡ„еҮ дёӘе®№еҷЁд№Ӣй—ҙе…ұдә«иө„жәҗпјҢдҫӢеҰӮipең°еқҖпјҢе…ұдә«еӯҳеӮЁзӯүдҝЎжҒҜгҖӮеҰӮжһңзӣҙжҺҘи°ғеәҰе®№еҷЁзҡ„иҜқпјҢйӮЈд№ҲеҮ дёӘе®№еҷЁеҸҜиғҪиҝҗиЎҢеңЁдёҚеҗҢзҡ„дё»жңәдёҠпјҢиҝҷж ·е°ұеўһеҠ дәҶзі»з»ҹзҡ„еӨҚжқӮжҖ§гҖӮ

1.2. Pod жҺ§еҲ¶еҷЁ вҖӢ

1.2.1. Deployment вҖӢ

ж— зҠ¶жҖҒеә”з”ЁйғЁзҪІгҖӮDeployment зҡ„дҪңз”ЁжҳҜз®ЎзҗҶе’ҢжҺ§еҲ¶Podе’ҢReplicasetпјҢ з®ЎжҺ§е®ғд»¬иҝҗиЎҢеңЁз”ЁжҲ·жңҹжңӣзҡ„зҠ¶жҖҒдёӯгҖӮ

1.2.2. Replicaset вҖӢ

зЎ®дҝқйў„жңҹзҡ„PodеүҜжң¬ж•°йҮҸгҖӮReplicaset зҡ„дҪңз”Ёе°ұжҳҜз®ЎзҗҶе’ҢжҺ§еҲ¶PodпјҢз®ЎжҺ§д»–д»¬еҘҪеҘҪе№Іжҙ»гҖӮ дҪҶжҳҜпјҢReplicaset еҸ—жҺ§дәҺDeploymentгҖӮ

еҸҜд»ҘзҗҶи§ЈжҲҗ Deployment е°ұжҳҜжҖ»еҢ…е·ҘеӨҙпјҢдё»иҰҒиҙҹиҙЈзӣ‘зқЈеә•дёӢзҡ„е·ҘдәәPodе№Іжҙ»пјҢзЎ®дҝқжҜҸж—¶жҜҸеҲ»жңүз”ЁжҲ·иҰҒжұӮж•°йҮҸзҡ„PodеңЁе·ҘдҪңгҖӮеҰӮжһңдёҖж—ҰеҸ‘зҺ°жҹҗдёӘе·ҘдәәPodдёҚиЎҢдәҶпјҢе°ұиө¶зҙ§ж–°жӢүдёҖдёӘPodиҝҮжқҘжӣҝжҚўе®ғгҖӮиҖҢ ReplicaSet е°ұжҳҜжҖ»еҢ…е·ҘеӨҙжүӢдёӢзҡ„е°ҸеҢ…е·ҘеӨҙгҖӮ

д»ҺK8SдҪҝз”ЁиҖ…и§’еәҰжқҘзңӢпјҢз”ЁжҲ·дјҡзӣҙжҺҘж“ҚдҪң Deployment йғЁзҪІжңҚеҠЎпјҢиҖҢеҪ“ Deployment иў«йғЁзҪІзҡ„ж—¶еҖҷпјҢK8S дјҡиҮӘеҠЁз”ҹжҲҗиҰҒжұӮзҡ„ ReplicaSet е’Ң PodгҖӮз”ЁжҲ·еҸӘйңҖиҰҒе…іеҝғ Deployment иҖҢдёҚж“Қеҝғ ReplicaSetгҖӮ

иө„жәҗеҜ№иұЎReplication ControllerжҳҜ ReplicaSet зҡ„еүҚиә«пјҢе®ҳж–№жҺЁиҚҗз”Ё Deployment еҸ–д»Ј Replication Controller жқҘйғЁзҪІжңҚеҠЎ

1.2.3. Daemonset вҖӢ

зЎ®дҝқжүҖжңүиҠӮзӮ№иҝҗиЎҢеҗҢдёҖзұ»PodпјҢдҝқиҜҒжҜҸдёӘиҠӮзӮ№дёҠйғҪжңүдёҖдёӘжӯӨзұ»PodиҝҗиЎҢпјҢйҖҡеёёз”ЁдәҺе®һзҺ°зі»з»ҹзә§еҗҺеҸ°д»»еҠЎгҖӮ

1.2.4. Statefulset вҖӢ

жңүзҠ¶жҖҒеә”з”ЁйғЁзҪІ

1.2.5. Job вҖӢ

дёҖж¬ЎжҖ§д»»еҠЎгҖӮж №жҚ®з”ЁжҲ·зҡ„и®ҫзҪ®пјҢJobз®ЎзҗҶзҡ„PodжҠҠд»»еҠЎжҲҗеҠҹе®ҢжҲҗе°ұиҮӘеҠЁйҖҖеҮәдәҶ

1.2.6. Cronjob вҖӢ

е‘ЁжңҹжҖ§и®ЎеҲ’жҖ§д»»еҠЎ

1.3. Label вҖӢ

ж ҮзӯҫпјҢжҳҜK8Sзү№иүІзҡ„з®ЎзҗҶж–№ејҸпјҢдҫҝдәҺеҲҶзұ»з®ЎзҗҶиө„жәҗеҜ№иұЎгҖӮ

LabelеҸҜд»Ҙйҷ„еҠ еҲ°еҗ„з§Қиө„жәҗеҜ№иұЎдёҠпјҢдҫӢеҰӮNodeгҖҒPodгҖҒServiceгҖҒ RCзӯүпјҢз”ЁдәҺе…іиҒ”еҜ№иұЎгҖҒжҹҘиҜўе’ҢзӯӣйҖүгҖӮ

дёҖдёӘLabelжҳҜдёҖдёӘ key-value зҡ„й”®еҖјеҜ№пјҢе…¶дёӯ key дёҺ value з”ұз”ЁжҲ·иҮӘе·ұжҢҮе®ҡгҖӮ

дёҖдёӘиө„жәҗеҜ№иұЎеҸҜд»Ҙе®ҡд№үд»»ж„Ҹж•°йҮҸзҡ„LabelпјҢеҗҢдёҖдёӘLabelд№ҹеҸҜд»Ҙиў«ж·»еҠ еҲ°д»»ж„Ҹж•°йҮҸзҡ„иө„жәҗеҜ№иұЎдёӯпјҢд№ҹеҸҜд»ҘеңЁеҜ№иұЎеҲӣе»әеҗҺеҠЁжҖҒж·»еҠ жҲ–иҖ…еҲ йҷӨгҖӮеҸҜд»ҘйҖҡиҝҮз»ҷжҢҮе®ҡзҡ„иө„жәҗеҜ№иұЎжҚҶз»‘дёҖдёӘжҲ–еӨҡдёӘдёҚеҗҢзҡ„LabelпјҢжқҘе®һзҺ°еӨҡз»ҙеәҰзҡ„иө„жәҗеҲҶз»„з®ЎзҗҶеҠҹиғҪгҖӮ

1.4. LabelйҖүжӢ©еҷЁ(Label selector ) вҖӢ

з»ҷжҹҗдёӘиө„жәҗеҜ№иұЎе®ҡд№үдёҖдёӘLabelпјҢ е°ұзӣёеҪ“дәҺз»ҷе®ғжү“дәҶдёҖдёӘж ҮзӯҫпјҢйҡҸеҗҺеҸҜд»ҘйҖҡиҝҮж ҮзӯҫйҖүжӢ©еҷЁ(Label selector) жҹҘиҜўе’ҢзӯӣйҖүжӢҘжңүжҹҗдәӣLabelзҡ„иө„жәҗеҜ№иұЎгҖӮ

ж ҮзӯҫйҖүжӢ©еҷЁзӣ®еүҚжңүдёӨз§ҚпјҡеҹәдәҺзӯүеҖје…ізі»(зӯүдәҺгҖҒдёҚзӯүдәҺ)е’ҢеҹәдәҺйӣҶеҗҲе…ізі»(еұһдәҺгҖҒдёҚеұһдәҺгҖҒеӯҳеңЁ)гҖӮ

1.5. Service вҖӢ

еңЁK8Sзҡ„йӣҶзҫӨйҮҢпјҢиҷҪз„¶жҜҸдёӘPodдјҡиў«еҲҶй…ҚдёҖдёӘеҚ•зӢ¬зҡ„IPең°еқҖпјҢдҪҶз”ұдәҺPodжҳҜжңүз”ҹе‘Ҫе‘Ёжңҹзҡ„(е®ғд»¬еҸҜд»Ҙиў«еҲӣе»әпјҢиҖҢдё”й”ҖжҜҒд№ӢеҗҺдёҚдјҡеҶҚеҗҜеҠЁ)пјҢйҡҸж—¶еҸҜиғҪдјҡеӣ дёәдёҡеҠЎзҡ„еҸҳжӣҙпјҢеҜјиҮҙиҝҷдёӘIPең°еқҖд№ҹдјҡйҡҸзқҖ Pod зҡ„й”ҖжҜҒиҖҢж¶ҲеӨұпјҢServiceе°ұжҳҜз”ЁжқҘи§ЈеҶіиҝҷдёӘй—®йўҳзҡ„ж ёеҝғжҰӮеҝөгҖӮ

K8Sдёӯзҡ„ Service е№¶дёҚжҳҜжҲ‘д»¬еёёиҜҙзҡ„вҖңжңҚеҠЎвҖқзҡ„еҗ«д№үпјҢиҖҢжӣҙеғҸжҳҜзҪ‘е…іеұӮпјҢеҸҜд»ҘзңӢдҪңдёҖз»„жҸҗдҫӣзӣёеҗҢжңҚеҠЎзҡ„Podзҡ„еҜ№еӨ–и®ҝй—®жҺҘеҸЈгҖҒжөҒйҮҸеқҮиЎЎеҷЁпјҢService дҪңз”ЁдәҺе“ӘдәӣPodпјҢжҳҜйҖҡиҝҮж ҮзӯҫйҖүжӢ©еҷЁжқҘе®ҡд№үзҡ„гҖӮ

еңЁK8SйӣҶзҫӨдёӯпјҢService еҸҜд»ҘзңӢдҪңдёҖз»„жҸҗдҫӣзӣёеҗҢжңҚеҠЎзҡ„ Pod зҡ„еҜ№еӨ–и®ҝй—®жҺҘеҸЈгҖӮе®ўжҲ·з«ҜйңҖиҰҒи®ҝй—®зҡ„жңҚеҠЎе°ұжҳҜ Service еҜ№иұЎгҖӮжҜҸдёӘServiceйғҪжңүдёҖдёӘеӣәе®ҡзҡ„иҷҡжӢҹIP (иҝҷдёӘIPд№ҹиў«з§°дёәCluster IP) пјҢиҮӘеҠЁе№¶дё”еҠЁжҖҒең°з»‘е®ҡеҗҺз«Ҝзҡ„Pod, жүҖжңүзҡ„зҪ‘з»ңиҜ·жұӮзӣҙжҺҘи®ҝй—® Service зҡ„иҷҡжӢҹIPпјҢServiceдјҡиҮӘеҠЁеҗ‘еҗҺз«ҜеҒҡиҪ¬еҸ‘гҖӮ

ServiceйҷӨдәҶжҸҗдҫӣзЁіе®ҡзҡ„еҜ№еӨ–и®ҝй—®ж–№ејҸд№ӢеӨ–пјҢиҝҳиғҪиө·еҲ°иҙҹиҪҪеқҮиЎЎ(Load Balance) зҡ„еҠҹиғҪпјҢиҮӘеҠЁжҠҠиҜ·жұӮжөҒйҮҸеҲҶеёғеҲ°еҗҺз«ҜжүҖжңүзҡ„жңҚеҠЎдёҠпјҢserviceеҸҜд»ҘеҒҡеҲ°еҜ№е®ўжҲ·йҖҸжҳҺең°иҝӣиЎҢж°ҙе№іжү©еұ•(scale)гҖӮ

иҖҢе®һзҺ° service иҝҷдёҖеҠҹиғҪзҡ„е…ій”®пјҢ е°ұжҳҜkube-proxyгҖӮ kube-proxyиҝҗиЎҢеңЁжҜҸдёӘиҠӮзӮ№дёҠпјҢзӣ‘еҗ¬API ServerдёӯжңҚеҠЎеҜ№иұЎзҡ„еҸҳеҢ–пјҢеҸҜйҖҡиҝҮд»ҘдёӢдёүз§ҚжөҒйҮҸи°ғеәҰжЁЎејҸ: userspace (еәҹејғ)гҖҒiptables (жҝ’дёҙеәҹејғ)гҖҒipvs (жҺЁиҚҗпјҢжҖ§иғҪжңҖеҘҪ)жқҘе®һзҺ°зҪ‘з»ңзҡ„иҪ¬еҸ‘гҖӮ

ServiceжҳҜK8SжңҚеҠЎзҡ„ж ёеҝғпјҢеұҸи”ҪдәҶжңҚеҠЎз»ҶиҠӮпјҢз»ҹдёҖеҜ№еӨ–жҡҙйңІжңҚеҠЎжҺҘеҸЈпјҢ зңҹжӯЈеҒҡеҲ°дәҶвҖңеҫ®жңҚеҠЎвҖқгҖӮжҜ”еҰӮжҲ‘д»¬зҡ„дёҖдёӘжңҚеҠЎAпјҢйғЁзҪІдәҶ3дёӘеүҜжң¬пјҢд№ҹе°ұжҳҜ3дёӘPod;еҜ№дәҺз”ЁжҲ·жқҘиҜҙпјҢеҸӘйңҖиҰҒе…іжіЁдёҖдёӘ Service зҡ„е…ҘеҸЈе°ұеҸҜд»ҘпјҢиҖҢдёҚйңҖиҰҒж“Қеҝғз©¶з«һеә”иҜҘиҜ·жұӮе“ӘдёҖдёӘPodгҖӮ

дјҳеҠҝйқһеёёжҳҺжҳҫпјҢдёҖж–№йқўеӨ–йғЁз”ЁжҲ·дёҚйңҖиҰҒж„ҹзҹҘеӣ дёәPodдёҠжңҚеҠЎзҡ„ж„ҸеӨ–еҙ©жәғгҖҒ K8S йҮҚж–°жӢүиө· Pod иҖҢйҖ жҲҗзҡ„IPеҸҳжӣҙпјҢеӨ–йғЁз”ЁжҲ·д№ҹдёҚйңҖиҰҒж„ҹзҹҘеӣ еҚҮзә§гҖҒеҸҳжӣҙжңҚеҠЎеёҰжқҘзҡ„PodжӣҝжҚўиҖҢйҖ жҲҗзҡ„IPеҸҳеҢ–гҖӮ

1.6. Ingress вҖӢ

Serviceдё»иҰҒиҙҹиҙЈK8S йӣҶзҫӨеҶ…йғЁзҡ„зҪ‘з»ңжӢ“жү‘пјҢйӮЈд№ҲйӣҶзҫӨеӨ–йғЁжҖҺд№Ҳи®ҝй—®йӣҶзҫӨеҶ…йғЁе‘ў?иҝҷдёӘж—¶еҖҷе°ұйңҖиҰҒIngressдәҶгҖӮ

IngressжҳҜж•ҙдёӘK8SйӣҶзҫӨзҡ„жҺҘе…ҘеұӮпјҢиҙҹиҙЈйӣҶзҫӨеҶ…еӨ–йҖҡи®ҜгҖӮIngressжҳҜ K8S йӣҶзҫӨйҮҢе·ҘдҪңеңЁOSIзҪ‘з»ңеҸӮиҖғжЁЎеһӢдёӢпјҢз¬¬7еұӮзҡ„еә”з”ЁпјҢеҜ№еӨ–жҡҙйңІзҡ„жҺҘеҸЈпјҢе…ёеһӢзҡ„и®ҝй—®ж–№ејҸжҳҜhttp/httpsгҖӮ

ServiceеҸӘиғҪиҝӣиЎҢз¬¬еӣӣеұӮзҡ„жөҒйҮҸи°ғеәҰпјҢиЎЁзҺ°еҪўејҸжҳҜip+portгҖӮIngressеҲҷеҸҜд»Ҙи°ғеәҰдёҚеҗҢдёҡеҠЎеҹҹгҖҒдёҚеҗҢURLи®ҝй—®и·Ҝеҫ„зҡ„дёҡеҠЎжөҒйҮҸгҖӮжҜ”еҰӮ:е®ўжҲ·з«ҜиҜ·жұӮhttp://www.qq.com: port ---> Ingress ---> Service ---> Pod

1.7. Name вҖӢ

з”ұдәҺK8SеҶ…йғЁпјҢдҪҝз”ЁвҖңиө„жәҗвҖқжқҘе®ҡд№үжҜҸдёҖз§ҚйҖ»иҫ‘жҰӮеҝө(еҠҹиғҪ)пјҢжүҖд»ҘжҜҸз§ҚвҖңиө„жәҗвҖқпјҢйғҪеә”иҜҘжңүиҮӘе·ұзҡ„вҖңеҗҚз§°вҖқгҖӮ

вҖңиө„жәҗвҖқжңүapiзүҲжң¬(apiversion) гҖҒзұ»еҲ«(kind)гҖҒе…ғж•°жҚ®(metadata) гҖҒе®ҡд№үжё…еҚ•(spec)гҖҒзҠ¶жҖҒ(status) зӯүй…ҚзҪ®дҝЎжҒҜгҖӮ

вҖңеҗҚз§°вҖқйҖҡеёёе®ҡд№үеңЁвҖңиө„жәҗвҖқзҡ„вҖңе…ғж•°жҚ®вҖқдҝЎжҒҜйҮҢгҖӮеңЁеҗҢдёҖдёӘ namespace з©әй—ҙдёӯеҝ…йЎ»жҳҜе”ҜдёҖзҡ„гҖӮ

1.8. Namespace вҖӢ

йҡҸзқҖйЎ№зӣ®еўһеӨҡгҖҒдәәе‘ҳеўһеҠ гҖҒйӣҶзҫӨи§„жЁЎзҡ„жү©еӨ§пјҢйңҖиҰҒдёҖз§ҚиғҪеӨҹйҖ»иҫ‘дёҠйҡ”зҰ»K8SеҶ…еҗ„з§ҚвҖңиө„жәҗ"зҡ„ж–№жі•пјҢиҝҷе°ұжҳҜNamespaceгҖӮ

NamespaceжҳҜдёәдәҶжҠҠдёҖдёӘK8SйӣҶзҫӨеҲ’еҲҶдёәиӢҘеҚғдёӘиө„жәҗдёҚеҸҜе…ұдә«зҡ„иҷҡжӢҹйӣҶзҫӨз»„иҖҢиҜһз”ҹзҡ„гҖӮдёҚеҗҢ Namespace еҶ…зҡ„вҖңиө„жәҗвҖқеҗҚз§°еҸҜд»ҘзӣёеҗҢпјҢзӣёеҗҢ Namespace еҶ…зҡ„еҗҢз§ҚвҖңиө„жәҗвҖқпјҢ вҖңеҗҚз§°вҖқдёҚиғҪзӣёеҗҢгҖӮ

еҗҲзҗҶзҡ„дҪҝз”ЁK8Sзҡ„ NamespaceпјҢеҸҜд»ҘдҪҝеҫ—йӣҶзҫӨз®ЎзҗҶе‘ҳиғҪеӨҹжӣҙеҘҪзҡ„еҜ№дәӨд»ҳеҲ°K8SйҮҢзҡ„жңҚеҠЎиҝӣиЎҢеҲҶзұ»з®ЎзҗҶе’ҢжөҸи§ҲгҖӮK8SйҮҢй»ҳи®ӨеӯҳеңЁзҡ„ Namespace жңү: defaultгҖҒ kube-systemгҖҒ kube-public зӯүгҖӮ

2. yamlзӨәдҫӢ вҖӢ

Deployment й…ҚзҪ®еҰӮдёӢпјҢжңү labelsгҖҒportsгҖҒreplicas зӯүгҖӮ

yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: deployment.nginx.the.test.com
spec:
  minReadySeconds: 0
  replicas: 1
  strategy:
    type: RollingUpdate
  selector:
    matchLabels:
      k8s-app: deployment.nginx.the.test.com
  template:
    metadata:
      labels:
        k8s-app: deployment.nginx.the.test.com
    spec:
      restartPolicy: Always
      terminationGracePeriodSeconds: 30
      nodeSelector: {}
      dnsPolicy: ClusterFirst
      containers:
        - name: nginx
          image: the/image:123-1
          ports:
            - name: container
              containerPort: 80
              protocol: TCP
          volumeMounts:
            - name: corefile
              mountPath: /data/corefile
              readOnly: false
            - name: logs
              mountPath: /data/logs
              readOnly: false
      volumes:
        - name: corefile
          hostPath:
            path: /data/corefile
        - name: logs
          hostPath:
            path: /data/logs
      imagePullSecrets:
        - name: nes-fe-secret

Service й…ҚзҪ®еҰӮдёӢпјҢжңү ports зӯүгҖӮ

yaml
apiVersion: v1
kind: Service
metadata:
  name: service-nginx-some-test-com
  labels: {}
  annotations: {}
spec:
  type: NodePort
  selector:
    k8s-app: deployment.nginx.the.test.com
  ports:
    - name: http
      port: 80
      protocol: TCP
      targetPort: container
      nodePort: 0

Ingress й…ҚзҪ®еҰӮдёӢпјҢжңү hostsгҖҒhttp зӯүгҖӮ

yaml
apiVersion: networking.k8s.io/v1beta1
kind: Ingress
metadata:
  name: ingress-the.test.com-0
  annotations:
    kubernetes.io/ingress.class: xxxcloud
    kubernetes.io/ingress.existLbId: xxx
spec:
  tls:
    - hosts:
        - the.test.com
      secretName: the.test.com.cert
  rules:
    - host: the.test.com
      http:
        paths:
          - path: /
            backend:
              serviceName: service-nginx-some-test-com
              servicePort: 80

# 公共库压缩合并

## 好处

对底层公共库主分支的MR，开始使用手动合并 + 压缩合并，去掉之前的自动合并钩子。好处如下：

1. 提交历史整洁，避免无效信息
2. `changelog` 清晰，无需再次手动整理

`github` 所有主流的开源项目都是使用的「压缩合并」。

## 步骤

新的合并步骤如下：

1. 选择压缩合并
2. 填写 MR 信息，精简、高度概括，如 `feat: add more application for network-v2`
3. 提交
